```java
@Service
public class PaymentCommandService {

    private final CommandGateway commandGateway;

    public String initiatePayment(InitiatePaymentRequest request) {
        String paymentId = UUID.randomUUID().toString();

        CreatePaymentCommand command = new CreatePaymentCommand(
            paymentId,
            request.getPaymentReference(),
            request.getAmount(),
            request.getCurrency(),
            request.getSender().getAccountNumber(),
            request.getRecipient().getAccountNumber()
        );

        commandGateway.sendAndWait(command);
        return paymentId;
    }
}
```

### 2.4 Controller Flow

```java
@PostMapping("/payments/initiate")
public ResponseEntity<ApiResponse<InitiatePaymentResponse>> initiate(
        @Valid @RequestBody InitiatePaymentRequest request,
        @RequestHeader(value = "Repeat-Flag", defaultValue = "false") boolean repeatFlag) {

    // 1. If repeatFlag → check idempotency cache
    if (repeatFlag) {
        Optional<InitiatePaymentResponse> cached = cacheService.get(request.getPaymentReference());
        if (cached.isPresent()) {
            return ResponseEntity.ok(ApiResponse.success(cached.get()));
        }
    }

    // 2. Generate UUID → send command → get paymentId
    String paymentId = paymentCommandService.initiatePayment(request);

    // 3. Cache the response
    InitiatePaymentResponse response = new InitiatePaymentResponse(paymentId, "INITIATED");
    cacheService.put(request.getPaymentReference(), response);

    // 4. Return 202 Accepted
    return ResponseEntity.status(HttpStatus.ACCEPTED)
        .body(ApiResponse.success(response, "Payment initiated successfully"));
}
```

### 3.2 PaymentAggregate

```java
@Aggregate
public class PaymentAggregate {

    @AggregateIdentifier
    private String paymentId;
    private String paymentReference;
    private BigDecimal amount;
    private String currency;
    private Status status;

    // No-arg constructor required by Axon
    protected PaymentAggregate() {}

    // ── Command Handlers ──

    @CommandHandler
    public PaymentAggregate(CreatePaymentCommand cmd) {
        // Validate invariants
        AggregateLifecycle.apply(new PaymentInitiatedEvent(
            cmd.getPaymentId(),
            cmd.getPaymentReference(),
            cmd.getAmount(),
            cmd.getCurrency(),
            cmd.getSenderAccountNumber(),
            cmd.getRecipientAccountNumber(),
            Instant.now()
        ));
    }

    @CommandHandler
    public void handle(ConfirmPaymentCommand cmd) {
        if (this.status != Status.DEBITED)
            throw new IllegalStateException("Cannot confirm payment in state: " + status);
        AggregateLifecycle.apply(new PaymentConfirmedEvent(
            cmd.getPaymentId(), Instant.now()
        ));
    }

    @CommandHandler
    public void handle(CancelPaymentCommand cmd) {
        AggregateLifecycle.apply(new PaymentCancelledEvent(
            cmd.getPaymentId(), cmd.getReason(), Instant.now()
        ));
    }

    // ── Event Sourcing Handlers (mutate state from events) ──

    @EventSourcingHandler
    public void on(PaymentInitiatedEvent e) {
        this.paymentId = e.getPaymentId();
        this.paymentReference = e.getPaymentReference();
        this.amount = e.getAmount();
        this.currency = e.getCurrency();
        this.status = Status.INITIATED;
    }

    @EventSourcingHandler
    public void on(PaymentConfirmedEvent e) {
        this.status = Status.CONFIRMED;
    }

    @EventSourcingHandler
    public void on(PaymentCancelledEvent e) {
        this.status = Status.CANCELLED;
    }
}
```

### 3.4 Outbox Pattern — Event Listener

```java
@Component
public class OutboxEventListener {
    private final OutboxRepository outboxRepository;
    private final ObjectMapper objectMapper;

    @EventHandler
    public void on(PaymentInitiatedEvent event) {
        writeOutbox(event.getPaymentId(), "PaymentInitiatedEvent", event);
    }

    @EventHandler
    public void on(PaymentConfirmedEvent event) {
        writeOutbox(event.getPaymentId(), "PaymentConfirmedEvent", event);
    }

    @EventHandler
    public void on(PaymentCancelledEvent event) {
        writeOutbox(event.getPaymentId(), "PaymentCancelledEvent", event);
    }

    private void writeOutbox(String aggregateId, String type, Object event) {
        OutboxEvent outbox = new OutboxEvent();
        outbox.setAggregateId(aggregateId);
        outbox.setEventType(type);
        outbox.setPayload(objectMapper.writeValueAsString(event));
        outbox.setPublished(false);
        outbox.setCreatedAt(Instant.now());
        outboxRepository.save(outbox);
    }
}
```

### 3.5 Outbox Pattern — Scheduled Relay

```java
@Component
public class OutboxRelay {
    private final OutboxRepository outboxRepository;
    private final KafkaTemplate<String, String> kafkaTemplate;

    @Scheduled(fixedDelay = 200)  // 200ms poll interval
    public void publishPendingEvents() {
        List<OutboxEvent> pending = outboxRepository
            .findByPublishedFalseOrderByCreatedAtAsc();

        for (OutboxEvent event : pending) {
            kafkaTemplate.send("payment.events", event.getAggregateId(), event.getPayload())
                .whenComplete((result, ex) -> {
                    if (ex == null) {
                        event.setPublished(true);
                        event.setPublishedAt(Instant.now());
                        outboxRepository.save(event);
                    }
                });
        }
    }
}
```

### 4.3 Rule Implementations

```java
// AmountThresholdRule
@Component
public class AmountThresholdRule implements RiskRule {
    @Value("${risk.amount.threshold:500000}")
    private BigDecimal threshold;

    @Override
    public RiskResult evaluate(PaymentInitiatedEvent event) {
        if (event.getAmount().compareTo(threshold) > 0) {
            return RiskResult.reject(
                "Amount " + event.getAmount() + " exceeds threshold " + threshold,
                "AMOUNT_THRESHOLD"
            );
        }
        return RiskResult.approve();
    }
}

// BlocklistRule
@Component
public class BlocklistRule implements RiskRule {
    @Value("${risk.blocklist.accounts}")
    private Set<String> blockedAccounts;

    @Override
    public RiskResult evaluate(PaymentInitiatedEvent event) {
        if (blockedAccounts.contains(event.getSenderAccountNumber()) ||
            blockedAccounts.contains(event.getRecipientAccountNumber())) {
            return RiskResult.reject("Account is on blocklist", "BLOCKLIST");
        }
        return RiskResult.approve();
    }
}

// VelocityRule — uses in-memory sliding window
@Component
public class VelocityRule implements RiskRule {
    @Value("${risk.velocity.max-per-minute:5}")
    private int maxPerMinute;

    private final ConcurrentHashMap<String, Deque<Instant>> senderHistory = new ConcurrentHashMap<>();

    @Override
    public RiskResult evaluate(PaymentInitiatedEvent event) {
        Deque<Instant> history = senderHistory
            .computeIfAbsent(event.getSenderAccountNumber(), k -> new ConcurrentLinkedDeque<>());

        Instant oneMinuteAgo = Instant.now().minus(Duration.ofMinutes(1));
        history.removeIf(ts -> ts.isBefore(oneMinuteAgo));
        history.addLast(Instant.now());

        if (history.size() > maxPerMinute) {
            return RiskResult.reject(
                "Velocity limit exceeded: " + history.size() + " txns in last minute",
                "VELOCITY"
            );
        }
        return RiskResult.approve();
    }
}
```

### 5.3 Debit Flow (Consumer)

```java
@Component
public class PaymentEventConsumer {

    @KafkaListener(topics = "payment.events", groupId = "account-service")
    @Transactional
    public void consume(PaymentInitiatedEvent event) {

        // 1. IDEMPOTENCY CHECK
        if (processedEventRepo.existsById(event.getPaymentId())) {
            log.info("Duplicate event {}, skipping", event.getPaymentId());
            return;
        }

        // 2. FIND SENDER ACCOUNT
        Account sender = accountRepo.findByAccountNumber(
            event.getSenderAccountNumber())
            .orElseThrow(() -> new AccountNotFoundException(
                "Account not found: " + event.getSenderAccountNumber()));

        // 3. CHECK BALANCE
        if (sender.getBalance().compareTo(event.getAmount()) < 0) {
            producer.send("account.events", new AccountDebitFailedEvent(
                event.getPaymentId(),
                event.getSenderAccountNumber(),
                "INSUFFICIENT_FUNDS",
                Instant.now()
            ));
            processedEventRepo.save(new ProcessedEvent(event.getPaymentId()));
            return;
        }

        // 4. DEBIT — with optimistic lock (@Version)
        sender.setBalance(sender.getBalance().subtract(event.getAmount()));
        accountRepo.save(sender);  // @Version prevents concurrent overwrites

        // 5. MARK AS PROCESSED + PUBLISH SUCCESS
        processedEventRepo.save(new ProcessedEvent(event.getPaymentId()));
        producer.send("account.events", new AccountDebitedEvent(
            event.getPaymentId(),
            event.getSenderAccountNumber(),
            event.getAmount(),
            Instant.now()
        ));
    }
}
```

### 6.2 PaymentSaga — Complete Implementation

```java
public class PaymentSaga {

    // Transient — injected by Axon, not serialized
    @Autowired
    private transient CommandGateway commandGateway;

    // Saga state — serialized between events
    private String paymentId;
    private boolean riskApproved = false;
    private boolean accountDebited = false;

    // ── STARTED by PaymentInitiatedEvent ──
    @StartSaga
    @SagaEventHandler(associationProperty = "paymentId")
    public void on(PaymentInitiatedEvent event) {
        this.paymentId = event.getPaymentId();
        // Just record — wait for risk + account results
    }

    // ── Risk approved ──
    @SagaEventHandler(associationProperty = "paymentId")
    public void on(RiskApprovedEvent event) {
        this.riskApproved = true;
        tryConfirm();
    }

    // ── Account debited ──
    @SagaEventHandler(associationProperty = "paymentId")
    public void on(AccountDebitedEvent event) {
        this.accountDebited = true;
        tryConfirm();
    }

    // ── BOTH succeeded → confirm payment ──
    private void tryConfirm() {
        if (riskApproved && accountDebited) {
            commandGateway.send(new ConfirmPaymentCommand(paymentId));
            SagaLifecycle.end();
        }
    }

    // ── Risk rejected → cancel payment ──
    @SagaEventHandler(associationProperty = "paymentId")
    public void on(RiskRejectedEvent event) {
        commandGateway.send(new CancelPaymentCommand(
            paymentId, "RISK_REJECTED: " + event.getReason()));
        SagaLifecycle.end();
    }

    // ── Account debit failed → cancel payment ──
    @SagaEventHandler(associationProperty = "paymentId")
    public void on(AccountDebitFailedEvent event) {
        commandGateway.send(new CancelPaymentCommand(
            paymentId, "INSUFFICIENT_FUNDS"));
        SagaLifecycle.end();
    }
}
```

> **Key:** `associationProperty = "paymentId"` routes events to the correct Saga instance. Every event must have a `getPaymentId()` method.

### 8.2 Dead Letter Topic Configuration

```java
@Bean
public DefaultErrorHandler errorHandler(KafkaTemplate<String, String> template) {
    DeadLetterPublishingRecoverer recoverer =
        new DeadLetterPublishingRecoverer(template);
    FixedBackOff backOff = new FixedBackOff(1000L, 3L);  // 3 retries, 1s apart
    return new DefaultErrorHandler(recoverer, backOff);
    // After 3 failures → message goes to {topic}.DLT
    // e.g. payment.events.DLT
}
```

## 9. Kafka Configuration

### 9.1 Producer Configuration

| Config Key | Value | Why |
|------------|-------|-----|
| `enable.idempotence` | `true` | Producer won't duplicate messages on retry. Kafka assigns sequence numbers per partition |
| `transactional.id` | `payment-producer-1` | Enables Kafka transactions. Epoch fencing prevents zombie producers |
| `acks` | `all` | All ISR replicas must ACK. No data loss on leader failure |
| `max.in.flight.per.connection` | `5` | Pipeline parallelism while idempotent producer preserves ordering |

### 9.2 Consumer Configuration

| Config Key | Value | Why |
|------------|-------|-----|
| `isolation.level` | `read_committed` | Consumer skips uncommitted transactional messages |
| `auto.offset.reset` | `earliest` | New consumer groups start from beginning — safe for projection rebuild |
| `enable.auto.commit` | `false` | Manual commit after idempotency check + business logic |

### 9.3 Topic Layout

| Topic | Partitions | Producers | Consumer Groups |
|-------|------------|-----------|-----------------|
| `payment.events` | 3 | Payment Service (Outbox) | risk-service, account-service, query-service, notification-service |
| `risk.events` | 3 | Risk Service | query-service, notification-service |
| `account.events` | 3 | Account Service | query-service, notification-service |
